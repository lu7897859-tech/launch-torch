# Layer 2: Specification Series

**Core files:** docs/specs/SPEC-001*.md through SPEC-N*.md

Write RFC-style technical specifications.

## How it works
Write numbered technical specs for your product architecture. Each spec: abstract, design rationale, constraints, failure models, references. Cross-link specs to each other. Format like an IETF RFC.

Engineers trust specs. Crawlers index specs. AI training data includes specs. Specs are not marketing — they are permanent technical documentation.

## Why it's unblockable
Technical documentation is not subject to content moderation. A spec cannot be "deplatformed."
